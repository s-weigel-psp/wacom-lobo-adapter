# Phase 3 — UI Design Contract

> Visual and interaction contract for the browser extension UI: Shadow DOM banner and options page.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none — vanilla JS + Web Platform APIs only |
| Icon library | none — text labels only, no icon dependency |
| Font | system-ui, -apple-system, sans-serif (Shadow DOM self-contained; no external font load) |

Source: RESEARCH.md Standard Stack — "No third-party npm packages are required or recommended."

---

## Surfaces

Two distinct UI surfaces with independent constraints.

### Surface 1: Shadow DOM Banner

Injected into the host page via `document.body.appendChild(host)`. The shadow root is `mode: 'closed'`. All CSS is inlined inside the shadow root as a `<style>` element. No host-page styles bleed in or out.

**Mounting point:** `<div id="wacom-bridge-banner-host">` appended to `document.body`, `position: absolute`, `z-index: 2147483647`.

**Positioning:** Anchored above the target DOM element. Host top = `rect.top + scrollY - 40px`. Host left = `rect.left + scrollX`. Recomputed on every banner state update.

**Banner height:** 40px (fixed). Used as the vertical offset so the banner sits directly above the target element without overlapping it.

**Banner width:** Auto — fits content. Minimum 220px.

### Surface 2: Options Page

Standard Chrome extension options page. Declared in `manifest.json` as `options_ui.page = "options.html"`, `open_in_tab: false`. Renders in Chrome's built-in settings panel. Host-page CSS isolation is not a concern here — the options page is a standalone HTML document.

## Color

### Banner State Colors

The banner uses semantic state colors. All colors are declared inside the Shadow DOM `<style>` block — never injected from the host page.

| State | Border | Background | Text |
|-------|--------|------------|------|
| Idle (not synced) | `#ccc` (`#cccccc`) | `#ffffff` | `#333333` |
| Synced (success) | `#4caf50` | `#e8f5e9` | `#1b5e20` |
| Area changed (warning) | `#ff9800` | `#fff3e0` | `#e65100` |
| Host not found (error) | `#f44336` | `#ffebee` | `#b71c1c` |

Source: RESEARCH.md Pattern 4 — established color values used verbatim.

### Color Contract (60/30/10)

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#ffffff` | Banner background (idle state), options page background |
| Secondary (30%) | `#f5f5f5` | Options page table rows (alternating), input field background |
| Accent (10%) | `#1976d2` | Action buttons ("Sync now", "Re-calibrate") background and focus ring |
| Destructive | `#f44336` | "Host not found" banner border, "Remove" button in options page |

Accent reserved for: action button backgrounds in the banner ("Sync now", "Re-calibrate"), options page "Add" button, options page "Save" button. Never used for text color, borders on non-interactive elements, or background fills.

### Banner Box Shadow

```css
box-shadow: 0 2px 8px rgba(0, 0, 0, 0.15);
```

Applied to all banner states. Provides elevation separation from the host page without relying on host-page context.

### Options Page

Options page uses the browser's default form styling. Custom CSS applied only for:
- Layout: flex column for the tuple list
- Destructive action: "Remove" button styled with `color: #f44336; border-color: #f44336`
- Primary action: "Add" / "Save" buttons styled with `background: #1976d2; color: #ffffff`

### Banner Component

**Tag:** `div.banner` inside `#wacom-bridge-banner-host` shadow root.

**States and DOM structure:**

#### Idle (not synced)
```

      <span class="banner-label">Wacom not synced</span>
```

#### Synced (success, auto-dismisses after 3 seconds)
```
    <div class="banner synced">
      <span class="banner-label">Wacom area synced</span>
    </div>
```
No button in the synced state. After 3 000ms the banner host is hidden (`host.style.display = 'none'`).

```
    <div class="banner area-changed">
      <span class="banner-label">PDF area changed</span>

```

#### Host not found (error)
```
    <div class="banner host-not-found">
      <span class="banner-label">Native host not found</span>
    </div>
```
No button — user must install the native host. No auto-dismiss.

## Interaction Contract

### Banner Lifecycle

| Trigger | State Transition | Banner Visibility |
|---------|-----------------|-------------------|
| Content script loads, element found, no prior sync | → idle | Shown |
| "Sync now" clicked | → (pending, button disabled) | Shown |
| `set_mapping` response: success | → synced | Shown, hidden after 3 000ms |
| `set_mapping` response: host error | → host-not-found | Shown, no auto-dismiss |
| ResizeObserver fires (after a successful sync) | → area-changed | Shown immediately |
| `window` resize event fires (after a successful sync) | → area-changed | Shown immediately |
| screenX/Y poll detects change (after a successful sync, 1s interval) | → area-changed | Shown immediately |
| "Re-calibrate" clicked | → (pending, button disabled) | Shown |
| Re-calibrate `set_mapping` response: success | → synced | Shown, hidden after 3 000ms |
| Re-calibrate `set_mapping` response: host error | → host-not-found | Shown, no auto-dismiss |

Source: CONTEXT.md D-05, D-06, D-07, D-08. Auto-dismiss duration: 3 000ms (RESEARCH.md A3).

### Staleness Detection Debounce

All three staleness triggers (ResizeObserver, `window` resize, screenX/Y poll) funnel through a single `onStale()` handler with a **300ms debounce**. This prevents banner flicker during active window-resize drags.

Source: RESEARCH.md A1, Pattern 5.

### Pending State (button disabled during in-flight request)

While a `NATIVE_COMMAND` message is in-flight to the service worker:
- The action button (`Sync now` or `Re-calibrate`) is `disabled` (HTML attribute).
- Button opacity: 0.6.
- Button cursor: `not-allowed`.
- Banner label text does not change during pending.

On response (success or error), button disabled state is removed and the banner transitions to the appropriate state.

### Page Visibility — Polling Pause

When `document.hidden === true` (tab backgrounded), the 1-second screenX/Y poll interval is paused (`clearInterval`). Polling resumes on `visibilitychange` if a prior sync exists.

Source: RESEARCH.md Pattern 5.

### Options Page Save Behavior

- "Save" button triggers `chrome.storage.sync.set({ targets: [...] })`.
- On success: display inline text "Saved." for 2 000ms, then remove.
- On error (storage quota exceeded): display "Save failed — storage quota exceeded."
- Empty `elementId` or `urlPattern` field blocks save and highlights the offending field in red.
- URL pattern validation: must start with `http://`, `https://`, or `file://`. If not, highlight in red and display inline hint: "Must start with http://, https://, or file://".

Note on destructive actions: The "Remove" button in the options page deletes a tuple immediately. No confirmation dialog is required — the data volume is low (typically 1-3 rows), the action is fully reversible by re-adding the tuple, and confirmation dialogs in small utility forms create friction without commensurate safety benefit. Enterprise users operate in a managed environment where IT controls default configuration.

## Accessibility Contract

Shadow DOM content is accessible to screen readers in closed mode (ARIA attributes on shadow host are visible to ATs).

| Requirement | Implementation |
|-------------|---------------|
| Banner role | `role="status"` on the inner `.banner` div — live region for AT announcement |
| Banner live region | `aria-live="polite"` on `.banner` div — announces state changes without interrupting |
| Action button label | Button text is the accessible label ("Sync now", "Re-calibrate") — no `aria-label` override needed |
| Disabled button | Use HTML `disabled` attribute — browsers communicate disabled state to ATs natively |
| Options inputs | `<label for="...">` element IDs associated to each input |
| Options page title | `<title>Wacom Bridge — Target Elements</title>` |
| Focus management | Options page: focus moves to the new row's first input on "Add target" |
