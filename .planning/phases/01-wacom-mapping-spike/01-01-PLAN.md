--- STEP 1: Locate PrefUtil.exe ---

Open an elevated PowerShell prompt and run:

```powershell
# Check both known names and common install locations
@(
    'C:\Program Files\Tablet\Wacom\PrefUtil.exe',
    'C:\Program Files\Tablet\Wacom\Wacom_TabletUserPrefs.exe',
    'C:\Program Files (x86)\Tablet\Wacom\PrefUtil.exe',
    'C:\Program Files (x86)\Tablet\Wacom\Wacom_TabletUserPrefs.exe'
) | ForEach-Object { [pscustomobject]@{ Path=$_; Exists=Test-Path $_ } } | Format-Table

# Also scan the Wacom install directory for all .exe files
Get-ChildItem 'C:\Program Files\Tablet\Wacom\' -Filter '*.exe' -ErrorAction SilentlyContinue | Select-Object Name, FullName
Get-ChildItem 'C:\Program Files (x86)\Tablet\Wacom\' -Filter '*.exe' -ErrorAction SilentlyContinue | Select-Object Name, FullName
```

Record the actual full path to the utility. This is $PREFUTIL_PATH for all subsequent steps.

--- STEP 2: Capture help output (verify flag syntax) ---

```powershell
$PREFUTIL_PATH = '<actual path from Step 1>'
& $PREFUTIL_PATH /?
& $PREFUTIL_PATH --help
& $PREFUTIL_PATH -help
```

One of these will show the supported flags. Look for export/import subcommands. Copy the full output to test-log.md.

--- STEP 3: Export BASELINE-A (current full-screen mapping) ---

Open the Wacom Tablet Properties application and confirm the tablet is currently set to full-screen (all-monitor) mapping. Then:

```powershell
# Create spike/ directory if it doesn't exist
New-Item -ItemType Directory -Path (Split-Path $PSCommandPath) -Force

$spikeDir  = 'C:\path\to\wacom-lobo-adapter\spike'   # adjust to repo location
$baselineA = Join-Path $spikeDir 'baseline-reference.xml'

$proc = Start-Process -FilePath $PREFUTIL_PATH `
            -ArgumentList '--export', $baselineA `
            -PassThru -Wait:$false
$proc | Wait-Process
Write-Host "Exit code: $($proc.ExitCode)"
Write-Host "Baseline A written to: $baselineA"
Get-Item $baselineA | Select-Object Length
```

Verify the file was created and is non-empty (should be several KB).

--- STEP 4: Change mapping via GUI and export BASELINE-B ---

1. Open Wacom Tablet Properties
2. Go to Mapping tab
3. Change "Screen Area" from "Full" to a specific region (e.g., left half of primary display)
4. Click OK / Apply and close

Then export the modified state:

```powershell
$baselineB = Join-Path $spikeDir 'baseline-modified.xml'

$proc = Start-Process -FilePath $PREFUTIL_PATH `
            -ArgumentList '--export', $baselineB `
            -PassThru
$proc | Wait-Process
Write-Host "Baseline B written to: $baselineB"
```

--- STEP 5: Diff the two XML files to find mapping tags ---

```powershell
[xml]$a = Get-Content $baselineA -Raw
[xml]$b = Get-Content $baselineB -Raw

# Convert both to sorted string and compare
$textA = $a.OuterXml -replace '><','>' + "`n<"
$textB = $b.OuterXml -replace '><','>' + "`n<"

# Write temp files for diff
$textA | Out-File "$env:TEMP\wacom-a.txt" -Encoding utf8
$textB | Out-File "$env:TEMP\wacom-b.txt" -Encoding utf8

# Show differing lines (requires any diff tool — fc, Compare-Object, or VS Code)
$linesA = Get-Content "$env:TEMP\wacom-a.txt"
$linesB = Get-Content "$env:TEMP\wacom-b.txt"
Compare-Object $linesA $linesB | Where-Object { $_.SideIndicator -ne '==' }
```

Look for lines that changed between baseline-A (full screen) and baseline-B (partial screen).
The element containing coordinates that changed is the screen-mapping element.

Record in test-log.md:
- The element name (e.g., `<ScreenMapping>`, `<MapArea>`, `<ScreenRect>`)
- All attribute names that changed (e.g., `Left`, `Top`, `Right`, `Bottom` or `X`, `Y`, `Width`, `Height`)
- Whether the XML uses a namespace (xmlns attribute on root element)
- The coordinate values in baseline-B (to verify they correspond to the region you set)

--- STEP 6: Discover Wacom service names ---

```powershell
Get-Service *wacom* -ErrorAction SilentlyContinue | Select-Object Name, DisplayName, Status
Get-Service *tablet* -ErrorAction SilentlyContinue | Select-Object Name, DisplayName, Status
Get-Service *wintab* -ErrorAction SilentlyContinue | Select-Object Name, DisplayName, Status
```

Record all matching service names and their status in test-log.md.

--- STEP 7: Test PrefUtil from NON-elevated prompt ---

Open a regular (non-admin) PowerShell prompt and attempt the same export:

```powershell
$proc = Start-Process -FilePath $PREFUTIL_PATH `

$proc | Wait-Process
Write-Host "Exit code: $($proc.ExitCode)"
Test-Path "$env:TEMP\wacom-nonadmin-test.xml"
```

Record whether the export succeeded without elevation in test-log.md.

The file already exists at spike/baseline-reference.xml from Step 3. Verify it is the full-screen version (NOT the modified one). If you made any test imports in between, re-export the full-screen baseline now.

Restore full-screen mapping via Wacom Tablet Properties UI before finishing.
