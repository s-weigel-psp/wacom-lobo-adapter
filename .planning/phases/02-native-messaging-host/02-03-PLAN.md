Namespace: http://wixtoolset.org/schemas/v4/wxs  (NOT the WiX 3 namespace)
Root element: Wix > Package (not Product as in WiX 3)
ProgramFiles6432Folder: installs to 64-bit C:\Program Files\ on 64-bit Windows
MajorUpgrade element: required for seamless future upgrades
RegistryKey/RegistryValue: used for HKLM entries (no RegistrySearch needed)
Build command: wix build installer/wacom-bridge.wxs -o wacom-bridge.msi

<!-- WOW6432Node (02-RESEARCH.md Pitfall 2) -->
A 64-bit WiX Package writing to SOFTWARE\Microsoft\Edge\NativeMessagingHosts\ covers 64-bit Edge.
For 32-bit Edge: also write to SOFTWARE\WOW6432Node\Microsoft\Edge\NativeMessagingHosts\
Both paths must be written for complete coverage. Chrome searches both paths natively.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MSI install → HKLM registry | MSI writes HKLM registry keys that Chrome/Edge read to locate the host; requires admin elevation at install time |
| HKLM registry → Chrome/Edge | Chrome/Edge reads the registry key default value to find the manifest JSON path; path must point to the correct file |
| manifest-chrome.json allowed_origins | Chrome/Edge enforces that only the listed extension IDs can connect to the host |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-03-01 | Spoofing | manifest-chrome.json / manifest-edge.json — allowed_origins | mitigate | allowed_origins must list only the specific published extension IDs; wildcard values are not permitted by the Chrome/Edge spec and must never appear; update PLACEHOLDER IDs to real IDs before production MSI build |
| T-02-03-02 | Tampering | HKLM registry key — manifest path | mitigate | Registry key is written to HKLM (not HKCU) so it requires admin elevation to modify; accidental or malicious modification of the key default value would redirect Chrome/Edge to a different manifest, potentially loading a different host binary; MSI MajorUpgrade repairs this on reinstall |
| T-02-03-03 | Tampering | installed manifest JSON files | accept | Manifest files in C:\Program Files\ require admin rights to modify; acceptable for the enterprise deployment scenario (domain admin controls the machine) |
| T-02-03-04 | Elevation of Privilege | MSI install — admin elevation requirement | accept | msiexec requires elevation to write to HKLM and Program Files; this is the correct and expected MSI behavior; D-15 explicitly documents admin elevation at install time; runtime binary does not require elevation |
| T-02-03-05 | Information Disclosure | installer source files in repository | accept | .wxs and manifest JSON files contain no secrets; placeholder extension IDs are not sensitive; the binary path (C:\Program Files\WacomBridge\) is public information |
</threat_model>
